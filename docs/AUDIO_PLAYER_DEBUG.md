# 音频播放器问题排查指南

## 常见问题

### 1. 点击播放后只有1秒噪音

**可能原因：**
- 预填充不足，音频流启动过早
- 缓冲区被快速消耗，没有持续填充
- 解码器或重采样器错误

**排查步骤：**

1. **检查预填充日志**
   ```
   查找日志中的 "开始预填充" 和 "音频流已启动"
   应该看到：
   - "开始预填充，最小样本数: XXXX"
   - "✅ 音频流已启动（预填充），缓冲区大小: XXXX"
   ```

2. **检查缓冲区状态**
   ```
   查找 "音频入队" 日志
   应该持续看到缓冲区在增长
   如果看到 "⚠️ 音频缓冲区不足"，说明填充速度跟不上消耗速度
   ```

3. **检查解码错误**
   ```
   查找 "音频重采样失败" 或 "发送音频包失败"
   如果有这些错误，说明解码有问题
   ```

### 2. 音频流没有稳定的输出

**可能原因：**
- 缓冲区被清空
- 播放状态被意外设置为 false
- 音频流被暂停

**排查步骤：**

1. **检查播放状态**
   ```
   查找 "位置更新" 日志
   应该看到位置持续增长
   如果位置停止更新，说明播放已停止
   ```

2. **检查缓冲区消耗**
   ```
   查找 "⚠️ 音频缓冲区不足" 警告
   如果频繁出现，说明：
   - 解码速度太慢
   - 缓冲区大小设置不合理
   - 音频流消耗速度太快
   ```

3. **检查 EOF 处理**
   ```
   查找 "解码器 EOF" 日志
   EOF 后应该继续播放缓冲区中的数据
   如果立即停止，说明 EOF 处理有问题
   ```

## 关键日志点

### 启动阶段
- `开始预填充，最小样本数: XXXX` - 预填充开始
- `✅ 音频流已启动（预填充）` - 流成功启动
- `⚠️ 预填充不足` - 预填充失败，需要等待

### 播放阶段
- `位置更新: X.XXs / X.XXs` - 位置正常更新
- `音频入队: take_len=XXX, buffered=XXX` - 数据正常入队
- `⚠️ 音频缓冲区不足` - 缓冲区不足警告

### 错误阶段
- `音频重采样失败` - 重采样错误
- `发送音频包失败` - 解码器错误
- `启动音频输出流失败` - 音频设备错误

## 调试技巧

### 1. 启用详细日志
在 `src-tauri/src/audio_player.rs` 中，所有关键操作都有日志：
- `log::debug!` - 调试信息
- `log::info!` - 重要事件（✅ 标记）
- `log::warn!` - 警告（⚠️ 标记）
- `log::error!` - 错误

### 2. 检查缓冲区大小
```rust
// 在 fill_audio_buffer 中添加日志
log::debug!("缓冲区状态: 请求={}, 可用={}, 弹出={}", 
    data.len(), buffer_len_before, samples_popped);
```

### 3. 检查位置计算
```rust
// 在位置更新时添加日志
log::debug!("位置计算: samples_played={}, buffer_samples={}, played_samples={}, pos={:.2}s",
    samples_played, buffer_samples, played_samples, pos);
```

## 常见修复方案

### 问题：预填充不足
**解决方案：**
1. 增加预填充量（修改 `min_prefill_samples`）
2. 等待更长时间再启动流
3. 检查音频文件是否正常

### 问题：缓冲区快速消耗
**解决方案：**
1. 增加缓冲区大小（`buffer_cap`）
2. 优化解码速度
3. 检查音频采样率是否过高

### 问题：EOF 后立即停止
**解决方案：**
- 已修复：EOF 后不再立即停止，而是等待缓冲区播放完

## 测试建议

1. **测试短音频（< 10秒）**
   - 检查是否能正常播放到结束
   - 检查位置是否准确

2. **测试长音频（> 1分钟）**
   - 检查是否能持续播放
   - 检查缓冲区是否稳定

3. **测试不同格式**
   - MP3, WAV, AAC 等
   - 检查是否有格式相关的问题
