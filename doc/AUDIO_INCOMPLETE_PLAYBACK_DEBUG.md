# 音频播放不完整问题排查指南

## 问题描述

直接播放只能播放到 123.04s，但跳转到中间再播放能播放到 122.26s，能听到最后一个字发音。

## 排查步骤

### 1. 查看音频时长信息

在日志中查找：

```
🎵 音频时长: XX.XX 秒 (原始 duration: XXXXX)
```

**关键信息：**

- 确认 `audio_duration` 是否准确
- 如果 `原始 duration` 是 `AV_NOPTS_VALUE` 或负数，说明文件元数据可能有问题

### 2. 查看数据包迭代器结束时的状态

在日志中查找：

```
📦 数据包迭代器结束，当前位置: XX.XXs / XX.XXs (进度: XX.X%)，已处理样本: XXXXX，缓冲区: XX 样本
```

**关键信息：**

- **当前位置**：数据包迭代器结束时的播放位置
- **已处理样本**：已解码并放入缓冲区的样本数
- **缓冲区样本**：当前缓冲区中剩余的样本数
- **进度百分比**：当前位置占总时长的百分比

**问题判断：**

- 如果当前位置 < 总时长 \* 0.99，说明数据包迭代器提前结束
- 如果已处理样本 < 预期样本数（总时长 \* 采样率），说明还有数据未处理

### 3. 查看 Flush 后的处理结果

在日志中查找：

```
✅ Flush 后处理了 X 帧，当前位置: XX.XXs / XX.XXs，已处理样本: XXXXX，继续循环
```

或

```
⚠️ Flush 后没有更多帧，当前位置: XX.XXs / XX.XXs，已处理样本: XXXXX，进入等待缓冲区播放逻辑
```

**关键信息：**

- **Flush 后处理的帧数**：解码器缓冲区中剩余的帧数
- **Flush 后的位置**：处理完所有帧后的位置
- **已处理样本**：处理完所有帧后的总样本数

**问题判断：**

- 如果 Flush 后位置 < 总时长 \* 0.99，说明还有数据未处理
- 如果 Flush 后没有更多帧，但位置 < 总时长，可能是数据包迭代器提前结束

### 4. 查看播放完成判断

在日志中查找：

```
🔍 播放完成检查: 位置=XX.XXs / XX.XXs (进度=XX.X%)，音频时钟=XX.XXs，已处理样本=XXXXX，缓冲区=XX 样本
```

**关键信息：**

- **位置**：当前播放位置（基于音频时钟或 samples_processed）
- **音频时钟**：基于实际播放样本数计算的时钟
- **已处理样本**：已解码的总样本数
- **样本比例**：已处理样本 / 预期样本数

**问题判断：**

- 如果样本比例 < 0.99，说明还有数据未处理
- 如果位置 < 总时长 \* 0.99，但判断为完成，说明判断逻辑有问题

### 5. 查看接近结尾的数据包

在日志中查找：

```
📦 处理接近结尾的数据包: PTS=XX.XXs (位置: XX.XXs / XX.XXs)
```

**关键信息：**

- **最后一个数据包的 PTS**：最后一个数据包的时间戳
- 如果最后一个数据包的 PTS < 总时长，说明数据包迭代器提前结束

## 可能的原因

### 1. 数据包迭代器提前结束

**症状：**

- 数据包迭代器结束时，当前位置 < 总时长 \* 0.99
- 最后一个数据包的 PTS < 总时长

**可能原因：**

- FFmpeg 数据包迭代器在文件末尾提前返回 `None`
- 文件元数据不准确，导致迭代器认为文件已结束

**解决方案：**

- 尝试使用 `ictx.packets()` 后，手动检查是否还有数据包
- 在数据包迭代器结束后，尝试继续读取（可能需要重新打开文件）

### 2. 播放完成判断过早

**症状：**

- 样本比例 < 0.99，但判断为完成
- 位置 < 总时长 \* 0.99，但判断为完成

**可能原因：**

- 判断逻辑的阈值（99%）太宽松
- 音频时钟计算不准确

**解决方案：**

- 调整判断逻辑，使用更严格的阈值（如 99.5%）
- 同时检查样本比例和位置，确保两者都接近总时长

### 3. 音频时长不准确

**症状：**

- `audio_duration` 与实际文件时长不符
- 原始 duration 是 `AV_NOPTS_VALUE` 或负数

**可能原因：**

- 文件元数据不准确
- FFmpeg 无法正确读取文件时长

**解决方案：**

- 使用 `ffprobe` 检查文件的实际时长
- 尝试从音频流获取时长（代码中已有此逻辑）

## 调试建议

1. **运行播放并查看完整日志**：

   ```bash
   # 在 Tauri 应用中播放音频文件，查看控制台日志
   # 或查看日志文件（如果配置了文件日志）
   ```

2. **对比直接播放和跳转播放的日志**：

   - 直接播放：查看数据包迭代器何时结束
   - 跳转播放：查看跳转后的数据包处理情况

3. **使用 ffprobe 验证文件信息**：

   ```bash
   ffprobe -v error -show_entries format=duration -show_entries stream=duration -of default=noprint_wrappers=1:nokey=1 your_audio_file.mp3
   ```

4. **检查文件完整性**：
   ```bash
   # 使用 ffmpeg 检查文件是否有问题
   ffmpeg -i your_audio_file.mp3 -f null -
   ```

## 下一步

根据日志信息，确定具体问题后：

1. 如果是数据包迭代器提前结束，需要修改数据包读取逻辑
2. 如果是播放完成判断过早，需要调整判断阈值
3. 如果是音频时长不准确，需要改进时长获取逻辑
